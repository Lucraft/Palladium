Items

Global Settings

These settings apply to ALL item types. Keep in mind that if fields are not required, you do NOT need to write them into your json.

Settings:

Setting Type Description Required Fallback Value
type Entry Item Type, each come with new different settings. Listed below on this page. false palladium:default
creative_mode_tab Entry ID of the creative mode tab the item is supposed to appear in. Fore more precise placements, check the "Custom Items" page on the wiki. Possible values: [minecraft:building_blocks, minecraft:colored_blocks, minecraft:combat, minecraft:food_and_drinks, minecraft:functional_blocks, minecraft:hotbar, minecraft:ingredients, minecraft:inventory, minecraft:natural_blocks, minecraft:op_blocks, minecraft:redstone_blocks, minecraft:search, minecraft:spawn_eggs, minecraft:tools_and_utilities, palladium:palladium_mods, palladium:technology] false /
components Entry Map of data components for this item. Check Minecraft Wiki false /

Example:

{"creative_mode_tab":"minecraft:decorations"}

GeckoLib Armor

Settings:

Setting Type Description Required Fallback Value
slot Entry The slot the item will fit in. Possible values: [helmet, chestplate, leggings, boots, body] true /
armor_material Entry Armor material, which defines certain characteristics about the armor. Open armor_materials.html for seeing how to make custom ones. Possible values: [minecraft:diamond, minecraft:gold, minecraft:leather, minecraft:turtle, minecraft:chainmail, minecraft:iron, minecraft:netherite, minecraft:armadillo, palladium:vibranium_weave] true /
armor_model Entry Path to geckolib model file. Required bones: [armorHead, armorBody, armorRightArm, armorLeftArm, armorRightLeg, armorLeftLeg, armorRightBoot, armorLeftBoot]. false /
armor_texture Entry Location of the armor texture. Can also use a dynamic texture using #. false /
armor_animations Entry ID of the animations that will be used. false /
armor_animation_controller Entry Names of controllers for the animation. false /
hide_second_player_layer Entry If enabled, the second player layer will be hidden when worn (only on the corresponding body part) false false

Example:

{"type":"geckolib:armor","slot":"chest","armor_material":"minecraft:diamond","armor_model":"palladium:test_model.geo.json","armor_texture":"example:textures/models/armor/example_armor.png","armor_animations":"palladium:animations/test_animation.animation.json","armor_animation_controller":[{"name":"controller_name","animation":"animation_name","transition_tick_time":10,"triggers":{"trigger_name":"animation_name"}}],"hide_second_player_layer":true}

Energy Item

Settings:

Setting Type Description Required Fallback Value
capacity Entry Max amount of energy the item can hold true /
max_input Entry Maximum amount of energy the item can be inserted with during one insertion. Using 0 makes the item not accept any energy true /
max_output Entry Maximum amount of energy the item can extract with during one withdrawal. Using 0 makes the item not extract any energy true /

Example:

{"type":"palladium:energy","capacity":500000,"max_input":1000,"max_output":1000}

Item

Default Item Type, you don't need to specify that you want this one, leaving 'type' out of the json will make it fall back to this one.

Example:

{"type":"minecraft:item"}

Armor

Settings:

Setting Type Description Required Fallback Value
slot Entry The slot the item will fit in. Possible values: [helmet, chestplate, leggings, boots, body] true /
armor_material Entry Armor material, which defines certain characteristics about the armor. Open armor_materials.html for seeing how to make custom ones. Possible values: [minecraft:diamond, minecraft:gold, minecraft:leather, minecraft:turtle, minecraft:chainmail, minecraft:iron, minecraft:netherite, minecraft:armadillo, palladium:vibranium_weave] true /
armor_renderer Entry Location of the armor renderer file. Doesn't need to be specified, it will automatically look for one in a path corresponding to the item's ID: A 'test:item' will look for the armor renderer file at 'assets/test/palladium/armor_renderers/item.json'. false /
openable Entry Marks the armor piece as openable. false false
opening_time Entry Determines the time the item needs for it to be fully opened. Leave at 0 for instant. Needs 'openable' to be enabled to take effect. false 0
opened_sound Entry Sound that is played when the suit has been fully opened. false /
closed_sound Entry Sound that is played when the suit has been fully closed. false /
opening_toggle_sound Entry Sound that is played when opening button has been pressed. false /

Example:

{"type":"minecraft:armor","slot":"chest","armor_material":"minecraft:diamond","armor_renderer":"test:item_renderer","openable":false,"opening_time":10,"opened_sound":"minecraft:item.armor.equip_leather","closed_sound":"minecraft:item.armor.equip_leather","opening_toggle_sound":"minecraft:item.armor.equip_leather"}

Shield

Settings:

Setting Type Description Required Fallback Value
use_duration Entry Amount of ticks the shield can be actively held for false 72000
repair_ingredient Entry The ingredient needed to repair the shield in an anvil. Can be null for making it non-repairable true /

Example:

{"type":"minecraft:shield","use_duration":72000,"repair_ingredient":{"tag":"minecraft:planks"}}

Bow

Settings:

Setting Type Description Required Fallback Value
velocity Entry Velocity multiplier for the shot projectile false 3.0
inaccuracy Entry Inaccuracy for the shot projectile false 1.0
use_duration Entry Amount of ticks the bow can be used for false 72000
projectiles Entry Item tag which contains all items that can be shot. By default all Minecraft arrows false minecraft:arrows
held_projectiles Entry Item tag which contains all items that can be shot by being in the off hand. Can be left out to fallback to the 'projectiles' option false /

Example:

{"type":"minecraft:bow","velocity":3.0,"inaccuracy":1.0,"use_duration":72000,"projectiles":"minecraft:arrows","held_projectiles":"minecraft:arrows"}
